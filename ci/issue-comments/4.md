Resolved in [Indexarr v0.3.2](https://github.com/AusAgentSmith-org/indexarr-rs/releases/tag/v0.3.2).

The Windows experience has been changed so Indexarr behaves like an installed desktop service rather than appearing to open and immediately close:

- The installer now asks which HTTP port Indexarr should use (default: `8080`), so you can choose another port when it is already occupied.
- The Windows service is registered with that selected port, keeping the service and browser URL in sync.
- Finishing setup opens the web interface at `http://127.0.0.1:<selected-port>/`.
- Launching `indexarr.exe` interactively also opens the configured local web interface once the server is ready.

For example, if you select port `8083`, use <http://127.0.0.1:8083/>. Please uninstall v0.3.1 and install the new [v0.3.2 Windows setup package](https://github.com/AusAgentSmith-org/indexarr-rs/releases/download/v0.3.2/indexarr-v0.3.2-windows-x86_64-setup.exe).

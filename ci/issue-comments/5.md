Resolved in [Indexarr v0.3.2](https://github.com/AusAgentSmith-org/indexarr-rs/releases/tag/v0.3.2).

The XML error mentioning `'src'` occurred because Prowlarr reached the Indexarr web application HTML instead of the Torznab XML endpoint. The route is now compatible with Prowlarr, and the setup documentation has been clarified.

Configure a **Generic Torznab** indexer in Prowlarr using separate values:

- **URL:** `http://your-indexarr-host:8080` (or the port configured for your installation)
- **API Path:** `/api/torznab`

Do not include `/api/torznab` in the URL field while leaving Prowlarr's default API Path in place, because Prowlarr appends the API Path to the URL. After upgrading Indexarr, save these values and run Prowlarr's indexer test again.

Relevant fix and documentation: [`02fcf23`](https://github.com/AusAgentSmith-org/indexarr-rs/commit/02fcf23).
